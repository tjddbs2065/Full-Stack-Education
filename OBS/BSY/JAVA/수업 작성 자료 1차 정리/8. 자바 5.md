##### 클래스 정의
```java
[접근지정자] class [클래스이름] {
    // 멤버 변수(필드)
    private int args = 0;

    // 생성자(Constructor)
    public [클래스이름]() {}

    // 메서드(Method)
    public [반환타입] [메서드이름]([매개변수]) { ... }
}```
- 클래스: 객체를 만들기 위한 설계도
- 구성 요소: 멤버 변수(필드), 생성자, 메서드

##### 캡슐화(Encapsulation)
- 정보 은닉(Information Hiding): 멤버 변수는 보통 `private`으로 선언 
	→ 해당 클래스의 외부에서 직접 접근을 차단.
- 접근자/설정자 메서드(`getter`,`setter`)를 통해 간접 접근
```java 
private String name;

public String getName() { return name; }
public void setName(String name) { this.name = name; }
```
- 메서드 네이밍은 멤버 변수명을 응용 → `getName()`, `setId()` 등
- 객체 생성을 원하지 않으면 생성자를 `private`으로 두고 생성자를 호출하지 못하게 경우도 있음.
	- ex) `Math`클래스 → 객체 필요 없음, 모든 기능이 `static` 메서드로 지원

##### 생성자(Constructor)
- 객체 생성 시 1회 호출(`new` 키워드 뒤에서 실행)
- 클래스 이름과 동일, 리턴 타입 없음.
- 따로 정의하지 않으면 JVM 기본 생성자(Default Constructor) 제공
	- 단, 생성자가 하나라도 있으면 기본 생성자는 제공되지 않음.
- 역할: 객체의 멤버 변수 초기화
- 생성자 오버로딩 가능: 매개변수를 다르게 하여 여러 방식의 초기화 지원
``` java
public User() { this.name = "Unknown"; }
public User(String name) { this.name = name; }
```

##### 메서드 오버로딩(Overloading)
- 메서드 이름은 같지만, 매개변수의 개수·타입이 다름
- 같은 동작을 다양한 자료형에 적용할 때 유용
``` java
public void print(int x) {}
public void print(String s) {}
```

##### 초기화(Initialization)
- 묵시적 초기화
	- 멤버 변수는 자료형의 기본값으로 자동 초기화.
	- 객체 생성 시 멤버 필드는 묵시적 초기화가 이루어진다.
- 명시적 초기화
	- 선언과 동시에 값을 지정하는 방식이다.
- 생성자 초기화
	- 생성자가 호출되면 값을 초기화 하는 방식이다.

##### 키워드
- this
	- 현재 객체 자기 자신을 참조(자신의 객체 주소)
	- 생성자 안에서 다른 생성자를 호출할 때 `this()`사용
- static
	- 객체와 무관하게 클래스 차원에서의 공유(공동사용)
	- static 변수: 모든 객체가 같은 값을 공유
	- static 메서드: 객체 생성 없이 호출 가능

##### 상속
- 클래스 간 공통된 멤버(필드, 메서드)를 재사용하기 위한 방법
- 여러 클래스의 중복 코드를 줄이고, 재사용성과 유지보수성을 높임
- 자식 클래스가 부모 클래스의 멤버를 물려받음

##### 상속 과정
- 여러 클래스에서 공통 멤버를 추출
- 추출된 멤버들을 하나의 부모 클래스에 모음.
- 기존 클래스는 부모 클래스를 상속받도록 변경
- 자바는 단일 상속만 허용(클래스는 하나의 부모만 상속 가능)

##### 생성자와 상속
- 자식 클래스 객체가 생성될 때 → 부모 생성자가 먼저 호출됨
- 이유: 부모의 필드 초기화 → 자식의 초기화 순서 필요
- JVM은 자식 생성자에 자동으로 super()를 삽입해 부모의 기본 생성자를 호출
	- 부모 클래스에 기본 생성자가 없으면, 반드시 `super(매개변수)`를 명시해야 함.
- `super()`는 생성자의 첫 번째 줄에서만 한 번 호출 가능

##### super 키워드
- `super()` → 부모 생성자 호출
- `super.멤버` → 부모 클래스의 멤버 변수/메서드 접근
- 단, `private`멤버는 상속되어 있어도 직접 접근 불가(캡슐화 원칙)

##### 메서드 오버라이딩(Overriding)
- 부모로부터 상속받은 메서드를 자식 클래스에서 재정의하는 것
- 조건
	- 메서드 이름, 매개변수, 반환 타입이 모두 같아야 함
	- 접근 제어자는 부모보다 더 좁을 수 없음
	- 부모 메서드가 `final`이면 오버라이딩 불가
	- 부모 메서드가 `static`이면 숨김은 가능하나 오버라이딩은 아님.
- 다형성의 핵심 → 부모 타입 참조 변수로 자식 객체를 다룰 때 자식의 오버라이딩 메서드가 실행됨.

##### 상속의 한계와 보완
- 자바는 다중 상속 불가 → 클래스 간 충돌 방지
- 보완 방법
	- 인터페이스: 다중 구현 가능
	- 합성: 상속보다는 합성을 선호하라(객체지향 원칙)
